Stock Dashboard | Python, Dash, Plotly, HTML/CSS

This is a stock analysis program made with Python using Dash. I was going to make a stock watchlist originally but decided against it. This project gave me an opportunity to improve at HTML/CSS, web app design, and using Python for data visualization. I also improved at pulling from API's and using Pandas and similar libraries to manipulate the data. There is a section for company metrics, a price graph, an analyst sentiment graph, a volume bubble graph, and a news module.

Odd Jobs | Unity, C#

This is a short horror game made in Unity. It was my first full 3D game and it changed forms a few different times. I learned some lessons about the importance of thorough pre-production planning and attempting to stick to the plan to avoid endless feature creep. I think some iteration while designing is a good thing but I wasted a lot time reworking things in hindsight. Main project components were level design, first person controller, interactions by raycast, horror design/events, and monster AI.

Game Jam / Class Projects

Lava Escape | UE4, Blueprints

Lava Escape is a puzzle platformer game made in Unreal for a gameplay programming course. You have to keep track of the room's layout and change elements between mushrooms that give you a double jump and clouds that disappear after a few seconds. This was my first Unreal game and helped me learn the editor and general workflow. I also learned a lot about level design and mindful placement and spacing of objects.

UATanks | Unity, C#

UATanks is a multiplayer tank arena game made in 5 weeks for a programming course. I learned about procedural generation and used it to randomize the map's layout. The enemies use state machines for decision-making and each enemy AI has unique behavior. I also added split-screen multiplayer and ran into some issues that ended up teaching me a lot about the Unity engine in general.

Thief in the Night | Unity, C#

I made this game for my first Itch.io Weekly Game Jam. The theme was Bandit and I chose the simple concept of defending treasure to make sure I finished the game within the time period. The biggest lesson I learned making it was to prototype the core gameplay loop to make sure the game is fun / engaging before making art and assets. I made that mistake and as a result the first iteration was too easy and boring. I ended up improving the thief AI and changing the game so that you only had four knives which had to be picked up on hit or miss. It's a better experience but could still benefit from new enemy types, levels, game mechanics, etc.
